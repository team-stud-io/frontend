# AI 튜터 프론트엔드 연동 검수

이 문서는 백엔드를 변경하지 않고 프론트엔드 연동을 확인하기 위한 체크리스트입니다. 백엔드는 백엔드 팀이 제공한 기존 실행 방법과 설정을 그대로 사용합니다.

## 실행

1. 백엔드를 기존 방식으로 실행합니다.
2. 이 폴더에서 `node server.js`를 실행합니다.
3. `http://127.0.0.1:5173`에 접속합니다.
4. 다른 API 주소를 사용할 때만 쿼리로 지정합니다.

   `http://127.0.0.1:5173/?apiBaseUrl=http://localhost:8080/api/v1`

깨끗한 상태로 다시 시작하려면 개발자 도구의 Application > Session Storage에서 `studio.aiTutorState`를 삭제하고 새로고침합니다.

## 정상 흐름

- 국어, 수학, 영어를 선택하고 서로 다른 시험일을 지정합니다.
- 같은 날짜에 두 과목도 지정해 봅니다.
- 각 과목은 시험 범위만 입력하고 나머지 선택값은 비운 상태로 진행해 봅니다.
- JPG 또는 PNG 프린트와 기출 이미지를 업로드합니다.
- 마지막 과목까지 저장한 뒤 전략 생성을 시작합니다.

Network 탭에서 다음 순서를 확인합니다.

1. 세션 생성
2. 선택한 과목 수만큼 과목 생성
3. 과목별 날짜가 분리된 `assignments` 저장
4. 과목별 범위, 스타일, 자료, 일정 저장
5. 이미지가 있을 때만 이미지 업로드
6. 모든 저장이 끝난 뒤 `strategies/generate` 1회 호출
7. 반환된 `job_id`로 상태 조회 반복
8. 완료 후 최신 전략, D-day, 과목별 전략 조회
9. 전략 저장과 플래너 반영 기록

`/complete`와 `/generation-status` 요청은 없어야 합니다.

## 복구와 오류

- 생성 중 새로고침하면 저장된 `job_id`로 상태 조회를 재개해야 합니다.
- 폴링이 90초를 넘으면 중단되고 다시 시도 버튼이 나타나야 합니다.
- 시간 초과나 일시적인 상태 조회 오류 후 다시 시도하면 기존 `job_id`를 조회해야 하며 `strategies/generate`를 다시 호출하면 안 됩니다.
- 작업 자체가 실패 상태이면 다시 시도할 때만 새 생성 작업을 요청합니다.
- 파일 선택 후 업로드 전에 새로고침하면 파일 재선택 안내가 표시되어야 합니다.
- 백엔드 연결 실패, 400/422, 404, 409, 500 계열 오류가 구분된 사용자 메시지로 표시되어야 합니다.
- 전략 저장 후 저장 버튼이 비활성화되어야 합니다.
- 동일한 플래너 대상을 다시 반영할 때 409를 정상적인 중복 상태로 처리해야 합니다.

## 문제 전달 형식

다음 정보만 기록해 전달합니다.

- 수행한 화면 단계
- 요청 URL과 HTTP 상태 코드
- 응답 오류 코드와 메시지
- 문제가 된 요청 필드
- 기대 동작과 실제 화면 동작

인증 헤더나 개인정보가 포함된 전체 HAR 파일은 공유하지 않습니다. 세부 과목 중복 조합은 백엔드 정책이 정해질 때까지 검수 대상에서 제외합니다.
